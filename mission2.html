Mission 2 : Exploitation avec Interface Low Code

Utilisation d'Access QBE pour interroger la base de données

Contexte

La mission 2 visait à exploiter la base de données KDou à travers une interface low code (Access QBE - Query By Example) pour créer des requêtes d'analyse sans programmation SQL directe.

Indicateur intéressant : Fournisseurs de viande et poisson

Énoncé

Identifier les fournisseurs qui proposent des produits dans les catégories "Viandes" ou "Poissons et fruits de mer" pour analyser les partenariats stratégiques de KDou dans ces segments clés.

Interprétation

Cette requête permet de visualiser les fournisseurs spécialisés dans les protéines animales, un segment essentiel du catalogue KDou. Elle aide à évaluer la diversité des sources d'approvisionnement.

Résultat

Société Pays
Sakura Imports Japon
Le Panier du Coin Canada
TanteGreta Handelsgruppe AG Allemagne

Bayou pice Traders États-Unis

Code SQL équivalent

SELECT DISTINCT Fournisseur.Societe, Fournisseur.Pays
FROM Categorie INNER JOIN (Fournisseur INNER JOIN Produit 
    ON Fournisseur.NoFour = Produit.NoFour)
ON Categorie.CodeCateg = Produit.CodeCateg 
WHERE Categorie.NomCateg = "Viandes" OR Categorie.NomCateg = "Poissons"
ORDER BY Fournisseur.Societe;

Analyse

On observe une diversité géographique des fournisseurs (Asie, Amérique du Nord, Europe), ce qui permet à KDou de ne pas dépendre d'une seule région pour ces produits critiques.
